HTML 5 Structure Tags:

The new tag in HTML don't provide an advantge in in terms of styling your pages instead they provide their standard for descibing a structure of the page.

This Standard makes it easier to build software that extracts infromation from the structured pages, so that it can be presented in an alternate way.

The first part of the structure tag is:-

Section tag: The top Level structure element of a page is the html element.

the next level is represented by head tag and body tag element

in HTML 4 the ony element available for dividing the pages into different section is the div tag. However HTML 5 provides a number of other tags for representing different section of a page.

Section tag: The Section element is used to define a generic section of a document. A Section tag is epected to include a heading tag for that section.

Header: The header element is a container for element that generally appears at the tp of the page and are carrie across an entire site of the website.

nav: HTML5 provides a nav tag that is supposed to contain a group of navigation links on he page.

article: The article element is another Sectioning element which is equivanlent t the section tag, with special semantic meaning. An article can be neted inside a section tag or a section tag can be nested inside an article tag.

Responsive Web Design

It is an Approch to web design that attempts to adapt a site design to look as good as possible on whatever device it is isplayed. The Ultimate goal of a responsive site is that it has 1 code of HTML, CSS and javascript and it can be displayed in any tpe of device.

RWD is important beacuse the number of browsers and the number of devics capable of displaying any website is growing day by day and everytime a new device is introduce to the market it adds new features to support and offfer changes the way it support the interesting features.

A good RWD include the following features:-

  1. Adjesting the conrtent so as to put the important things first for each device first.
  2. Recognising the critical content for a page and making sure that everyone who visites the page can access that information.
  3. Providing alternatives for the element that do not work on small screen.
  4. Keeping the bandwidht case whie designing a website.
  5. Usig technology that is a propriate and provides options for critical features.

Points should be remember to design a mobile web.

  1. limit your design to first column
  2. limit the navigation choice
  3. limit the file size
  4. limit the URL lenght
  5. limit the use of frame and tables

Planning a responsive Website

  1. Check your analytics
  2. Try to visualize the site using your own mobile phone.
  3. Design what content is critical.
