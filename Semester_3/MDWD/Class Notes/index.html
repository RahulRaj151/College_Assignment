URL

URL are refered as inform resource locator which are unique address for information on the internet.

                 host name
                ----------   
        https://google.com/home/www/file.html
        -----              ------------------
        protocal               path of file

Most of the URL's generally contain 3 parts

  1. Protocal
    2. The protocol is the way in which the page is accesed that means the way of communication your browser uses to get a file. eg; HTTP and FTP
  2. Host name
    3. The host name is the address of the computer on which the information is stored. the same host name can support more then 1 protocol. The hostname may sometime include a quote no. eg: 
    /google.com/500/
  3. path of he file
    4. a directory the path refers to the location of the file on the host system.

After a path some URL includes a query. the query is seperated from the rest of th URL by a question marl(?). The query is amde up of name and value pair which are separated by '&' symbol. There is '=' sign between name and value

representaion of special character

  • %20 -> space
  • %3f -> ?
  • %2f -> /
  • %2b -> space

Special character in URL is anything that is not a uppercase alphabet or lowercase alphabet or the digits from 0-9 and some symbols like '$' , '/' , '_' , .

Different types of URL

HTTP URL: HTTP URL are the most common type of the URL which is used by www to communicate with the web browsers. If he URL ends with a '/' it is considered as a directory name. a File name do not end with '/'

Anonymous FTP : FTP URL is used to locate a file in FTP server. Anonymous FTP allows you to login using Anonymous as the login id and your email address as the password eg: 

ftp://myhost.com/

Non-Anonymous FTP: Int this form of URL, the user name part is your ligin Id on the server and the password is th accounts password. eg: 

ftp://username:password@

File URL : The File URL are used to refer the files which are int he local disk eg: 

file:///dir1/dir2/file

Mailto URL : Mailto URL are used to send e-mails. If the Browser supports the mail to URL then when we Click on a link that contian an email, the browser will open a new out going email in your defaut email application eg: 

mailto: xyz@gmail.com?cc=abc@gmail.com&subject=hello&body=text
